# Linux File System

All files and directories in Linux are organized under the root directory **"/"**. Even if data is stored on different physical devices, it is presented as a single directory hierarchy. Some directories are always present, while others may or may not exist depending on the system configuration.

| Directory   | Purpose                                                                 |
| ----------- | ----------------------------------------------------------------------- |
| /           | Root directory containing the entire filesystem hierarchy               |
| /bin, /sbin | Essential system commands and utilities                                 |
| /var        | Variable data such as logs, cache files, and temporary application data |
| /dev        | Virtual device files representing hardware and system devices           |
| /etc        | System and application configuration files                              |

## File Permissions

Every file and directory has permissions that can be modified.

* **r (read)** = 4
* **w (write)** = 2
* **x (execute)** = 1

The command **ls -l** can be used to display additional information about files, including permissions, ownership, and other metadata.

## What Are Configuration Files?

Configuration files are text files that store settings for various system components and applications. They are commonly located in the following directories:

* **/etc/** — the primary directory for system-wide and application configuration files.
* **~/.config/** — directory containing user-specific configuration files.

| Service                         | Path                                                                 |
| ------------------------------- | -------------------------------------------------------------------- |
| Network interface configuration | /etc/network/interfaces<br>/etc/netplan/*.yaml                       |
| Web server configuration        | /etc/httpd/conf/httpd.conf (Apache)<br>/etc/nginx/nginx.conf (Nginx) |
| User and permission management  | /etc/passwd<br>/etc/shadow<br>/etc/group                             |
| Kernel configuration            | /etc/sysctl.conf                                                     |
| SSH configuration               | /etc/ssh/sshd_config                                                 |

Configuration files are typically plain text files and can be edited using text editors such as **nano**, **vim**, or **vi**.

## Examples

### Network Configuration

Changing the IP address in the **/etc/network/interfaces** file:

```bash
auto eth0
iface eth0 inet static
    address 192.168.1.10
    netmask 255.255.255.0
    gateway 192.168.1.1
```

### Web Server Configuration

Changing the document root for a web server in the **/etc/nginx/sites-available/default** file:

```nginx
server {
    listen 80;
    server_name example.com;
    root /var/www/example;
    index index.html;
}
```

After making changes, verify the configuration syntax and restart Nginx.

### System Parameter Configuration

Changing kernel parameters in the **/etc/sysctl.conf** file:

```bash
net.ipv4.ip_forward=1
```

Apply the changes:

```bash
sysctl -p
```

This command reloads the parameters from **/etc/sysctl.conf** without requiring a system reboot.


# Linux Logs

Every log message is generated as the result of an event occurring within the operating system. Each event is assigned a priority based on its severity. Linux defines the following log levels:

1. **emerg** — Emergency; the highest priority.
2. **alert** — Alert.
3. **crit** — Critical event.
4. **err** — Error.
5. **warn** — Warning.
6. **notice** — Notice.
7. **info** — Informational message.
8. **debug** — Debugging information.

> Linux logs are collected by the **rsyslog** and **systemd-journald** services. They operate independently of one another and are included in most modern Linux distributions.

System log files are typically stored in the **/var/log/** directory as plain text files. Depending on the type of event, messages are written to different log files.

**/var/log/** → log files

Some common log locations include:

* **/var/log/httpd/** — Contains HTTP request, response, and error logs.
* **/var/log/cron** — Stores events related to cron jobs.
* **/var/log/auth.log** and **/var/log/secure** — Store authentication-related logs.
* **/var/log/kern.log** (or **/var/log/kern**) — Stores kernel-related events.

Common locations for Apache logs are:

* **/var/log/apache/**
* **/var/log/httpd/**

Each line in a log file represents a separate message generated by an application or system service. Regardless of the source, log entries generally follow the same format and consist of five parts:

1. **Timestamp** — the date and time when the message was recorded.

   * Example: `Feb 12 06:18:33`

2. **Hostname** — the name of the computer that generated the message.

   * Example: `vds`

3. **Program or service name** — the application or service associated with the message.

   * Example: `sshd`

4. **Process ID (PID)** — the identifier of the process that generated the message.

   * Example: `[653]`

5. **Message text** — a description of the event.

   * Example:
     `Accepted password for mihail from 188.19.42.165 port 2849 ssh2`
